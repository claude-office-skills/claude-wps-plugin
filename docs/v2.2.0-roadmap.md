# Claude for WPS · v2.2.0 大版本规划

> **版本号：** v2.2.0  
> **主题：** Workflow Engine · 跨 App 自动化 · 触发器 · 渠道输出  
> **兼容性：** 完全向后兼容 v2.1.0，零破坏性变更  
> **参考：** OpenClaw Gateway 架构 + 设计文档 Phase 4/5

---

## 一、核心目标

将插件从"对话 → 执行"的封闭循环，升级为通用的自动化平台：

```
v2.1.0（当前）：用户输入 → AI 生成代码 → WPS 执行
                                    ↓
v2.2.0（目标）：
  任意触发 → Workflow 规划 → 跨 App 动作 → 任意渠道输出
  ─────────   ─────────────   ──────────   ──────────────
  手动输入    .workflow.yaml   wps.*        插件对话
  定时 Cron  Plan 模式步骤     ai.*         飞书 Webhook
  Webhook    条件分支          system.*     邮件 SMTP
  文件变化    用户审批门        file.*       HTTP Webhook
                               notify.*     本地文件
```

## 二、新增模块（全部加法，不修改现有代码）

### 模块 1：Heartbeat 心跳机制
**文件：** `lib/heartbeat.js`  
**目标：** Proxy 常驻运行，UI 感知状态，断线自恢复

```
proxy-server.js
  └── heartbeat.js
        ├── 每 30s 写 ~/.claude-wps/heartbeat.json
        ├── GET /health 返回增强版状态
        └── 插件 UI 每 15s 轮询，断连显示重连态
```

### 模块 2：Workflow Engine
**文件：** `lib/workflow-engine.js`  
**目标：** 解析执行 .workflow.yaml，支持条件分支、用户审批门

```yaml
# skills/workflows/xxx/workflow.yaml
name: my-workflow
trigger:
  type: manual | cron | webhook
  schedule: "0 9 * * 1-5"   # cron 专用
steps:
  - id: step1
    action: wps.readRange
  - id: step2
    action: ai.chat
    condition: $step1.rows > 0
  - id: step3
    action: notify.feishu
    params:
      message: "{{ $step2.result }}"
```

### 模块 3：Action Registry 扩展
**文件：** `lib/actions/system.js`, `lib/actions/file.js`, `lib/actions/notify.js`  
**目标：** 在现有 wps.*/ai.*/user.* 基础上新增三类动作

| 动作 | 描述 |
|------|------|
| `system.run` | 执行白名单 shell/osascript 命令 |
| `system.screenshot` | 截图 |
| `file.read` | 读取本地文件（CSV/PDF/JSON/XLSX） |
| `file.write` | 写入/导出文件 |
| `file.open` | 用默认 App 打开文件 |
| `http.get/post` | 调用外部 HTTP API |
| `notify.feishu` | 发飞书 Webhook 消息 |
| `notify.email` | 发邮件（SMTP 或 Mail.app） |
| `notify.webhook` | 通用 HTTP 通知 |

### 模块 4：Trigger 触发引擎
**文件：** `lib/trigger-engine.js`  
**目标：** Proxy 启动时扫描 workflows/，自动注册 Cron 定时任务

```javascript
// 触发类型
manual    // 用户在对话中触发（/workflow xxx）
cron      // node-cron 定时触发
webhook   // POST /trigger/:workflowName
file-watch // 本地文件变化触发（chokidar）
```

### 模块 5：Channel 输出层
**文件：** `lib/channels/` 目录  
**目标：** 统一的"发送到渠道"抽象，渠道配置写在用户本地

```
lib/channels/
  ├── index.js      # 渠道路由
  ├── feishu.js     # 飞书 Webhook
  ├── email.js      # nodemailer SMTP
  └── webhook.js    # 通用 HTTP
```

用户配置（`~/.claude-wps/channels.json`，不进 Git）：
```json
{
  "feishu": { "webhookUrl": "https://open.feishu.cn/..." },
  "email": {
    "smtp": { "host": "smtp.gmail.com", "port": 587 },
    "user": "me@gmail.com",
    "passwordEnvVar": "CLAUDE_WPS_EMAIL_PASS"
  }
}
```

## 三、新增 API 端点（不修改现有端点）

```
POST /workflow/start       —— 手动启动工作流
GET  /workflow/status/:id  —— 查询工作流执行状态
POST /workflow/approve/:id —— 用户审批门确认
POST /trigger/:name        —— Webhook 触发指定工作流
GET  /triggers             —— 列出所有已注册触发器
GET  /health               —— 增强版健康检查（含心跳信息）
```

5. **Channel 输出层**（1天）— 飞书 + 邮件 + 通用 Webhook
6. **版本号 & 构建**（0.5天）— bump 到 v2.2.0
